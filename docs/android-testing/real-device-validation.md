# REAL DEVICE VALIDATION REQUIRED

当前自动化只证明本地逻辑和 Android API 集成可编译，不能冒充真实 NotificationListenerService 设备证据。

## 检查表

1. 安装 Debug APK。
2. 首次启动阅读隐私说明并选择“发现后让我确认”。
3. 在系统设置开启时刻通知访问。
4. 只白名单一个测试应用。
5. 让该应用产生真实快递通知。
6. 确认草稿出现且原文没有写入数据库/日志。
7. 确认草稿并检查同一快递后续通知更新原实体。
8. 验证到站进入待取列表，取件码默认遮挡。
9. 通过设备凭据查看完整取件码。
10. 查询“我要拿快递了”，随后标记已取件。
11. 杀死并重启应用，确认状态仍存在。
12. 撤销系统通知访问，确认后续通知不再捕获。
13. 清除全部数据并检查数据库无取件码残留。

证据必须标注设备型号、Android 版本、APK SHA-256、测试时间和操作者。任何合成通知都标记 `DEBUG TEST SOURCE`。
