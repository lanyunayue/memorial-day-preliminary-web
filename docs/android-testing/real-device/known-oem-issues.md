# Known OEM Issues

> Template for documenting OEM-specific (manufacturer-specific) issues encountered during real-device testing of the alpha2 build.
>
> Fill in one section per OEM / ROM. Mark severity and whether a workaround exists.

---

## How to use this document

1. Add a section for each device manufacturer / ROM you test on.
2. Record the exact device model, ROM version, and Android API level.
3. For each issue, describe the symptom, the expected behavior, the actual behavior, the root cause (if known), and any workaround.
4. Link to the corresponding result JSON (`notification-listener-result.json`, `permission-revoke-result.json`, etc.) and any screenshots.
5. Severity legend: `BLOCKER` — blocks core functionality; `MAJOR` — significant degradation; `MINOR` — cosmetic or edge case.
