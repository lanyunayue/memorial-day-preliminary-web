# Failure And Pivot Criteria

Shrink or pivot if real evidence shows one or more repeated patterns:

- Participants treat the product only as a plain note.
- Waiting For and Commitment are almost unused.
- Multi-intent drafts add more work than they remove.
- Participants prefer the system calendar for the same task.
- There is no voluntary return within seven days.
- Next action explanations are not understood.
- Recognition errors prevent real input.
- Web entry friction is unacceptable.

Possible decisions are narrower ICP, narrower scenario, faster capture entry, a dedicated Waiting For tool, or pause. Adding unrelated features is not a response to failed validation.
