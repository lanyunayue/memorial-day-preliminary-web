# 真人试用方案：过载恢复机制验证

> **HUMAN OVERLOAD PILOT REQUIRED**
>
> 本方案涉及真人参与者的数据收集。所有数据收集活动必须在获得参与者书面知情同意后方可开始。本方案不可仅凭模拟数据或合成数据得出结论。

---

## 1. 研究目标

验证以下产品假设（H-001 至 H-010），评估"过载恢复"机制是否真正帮助用户在认知负荷过高时做出可持续的自我调节决策，而非产生抗拒或说教感。

### 假设清单

| 编号 | 假设描述 | 验证指标 |
|------|----------|----------|
| H-001 | 用户能在 30 秒内完成首次签到理解 | 首次签到耗时 checkInDurationMs < 30000 |
| H-002 | 负荷解释页面被至少 60% 的用户主动查看 | viewedLoadExplanation = true 的比例 >= 60% |
| H-003 | 降载选项被至少 40% 的用户打开查看 | openedDeLoad = true 的比例 >= 40% |
| H-004 | 建议接受率（accepted）不低于 30% | suggestionDecision = accepted 的比例 >= 30% |
| H-005 | 建议修改率（modified）不低于 15% | suggestionDecision = modified 的比例 >= 15% |
| H-006 | 建议拒绝率（rejected）不超过 55% | suggestionDecision = rejected 的比例 <= 55% |
| H-007 | 撤销使用率（usedUndo）不超过 20% | usedUndo = true 的比例 <= 20% |
| H-008 | 次日返回率不低于 50% | returnedNextDay = true 的比例 >= 50% |
| H-009 | 用户认为建议"不说教"的比例不低于 70% | feltPreachy = false 的比例 >= 70% |
| H-010 | 用户认为"有帮助"且"愿意继续使用"的比例不低于 40% | foundHelpful = true 且 willingToContinue = true 的比例 >= 40% |

## 2. 参与者

### 2.1 人数
- 目标人数: 10-15 名
- 最低有效人数: 10 名（低于此数无法得出统计可信结论）

### 2.2 年龄范围
- 18-28 岁目标用户

### 2.3 招募标准

#### 包含标准（全部满足）
1. 年龄 18-28 岁
2. 拥有个人 Android 手机（API 26+ / Android 8.0+）
3. 每日手机使用时间超过 4 小时
4. 自述在过去一个月内感到过"信息过载"或"任务太多处理不过来"
5. 能够阅读和理解中文
6. 愿意连续 7-14 天每日使用应用并记录反馈
7. 已签署知情同意书

#### 排除标准（满足任一即排除）
1. 正在接受临床心理治疗或服用影响认知功能的药物
2. 参与过本产品的前期设计、开发或测试工作
3. 无法保证连续 7 天的基本使用
4. 手机系统版本低于 Android 8.0
5. 年龄不在 18-28 岁范围内
6. 无法理解知情同意书内容

## 3. 时长

- 最短周期: 7 天
- 最长周期: 14 天
- 建议周期: 10 天
- 每日使用时长: 无强制要求，依赖自然使用行为
- 每日数据记录: 由 OVERLOAD_RESEARCH_MODE 自动捕获，无需手动填写

---

## 4. 数据收集方式

### 4.1 OVERLOAD_RESEARCH_MODE

数据收集通过应用内的 `OVERLOAD_RESEARCH_MODE` BuildConfig 字段控制。

- **构建配置**: `build.gradle.kts` 中 `buildConfigField("boolean", "OVERLOAD_RESEARCH_MODE", "true")`（试用版本设为 true）
- **触发条件**: 仅在 `OVERLOAD_RESEARCH_MODE = true` 时，应用才记录试用数据
- **存储位置**: 所有数据仅存储在参与者设备本地，不上传任何服务器
- **数据格式**: 符合 `pilot-data-schema.json` 定义的 JSON Schema
- **导出方式**: 参与者在试用结束后通过应用内"导出遮挡后的本地数据"功能导出 JSON 文件，手动发送给研究者

### 4.2 数据收集原则
- 仅收集行为指标和主观评价，不收集内容本身
- 所有签到文本、身体状态描述等原始内容均不记录
- 参与者通过匿名编码（participantCode）标识，不关联真实身份
- 数据导出前经过应用内置的掩码处理

---

## 5. 每日记录项

每个签到会话（session）记录以下字段:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| participantCode | string | 匿名编码（如 P01、P02...） |
| sessionId | string | 会话唯一标识 |
| date | string (ISO 8601) | 会话日期 |
| firstUse | boolean | 是否为首次使用 |
| checkInDurationMs | number | 签到耗时（毫秒） |
| viewedLoadExplanation | boolean | 是否查看负荷解释 |
| openedDeLoad | boolean | 是否打开降载选项 |
| suggestionDecision | enum | 建议决策: accepted / modified / rejected / not_shown |
| usedUndo | boolean | 是否使用撤销 |
| returnedNextDay | boolean | 次日是否返回使用 |
| feltPreachy | boolean | 用户认为建议是否说教 |
| foundHelpful | boolean\|null | 用户认为是否有帮助（可为 null 表示未回答） |
| willingToContinue | boolean\|null | 用户是否愿意继续使用（可为 null 表示未回答） |

### 记录时机
- `checkInDurationMs`: 从签到页面加载到签到提交的时间差
- `viewedLoadExplanation`: 用户点击"查看负荷解释"时记录
- `openedDeLoad`: 用户点击"降载"入口时记录
- `suggestionDecision`: 用户在建议卡片上点击"接受"/"修改"/"拒绝"时记录
- `usedUndo`: 用户点击撤销按钮时记录
- `returnedNextDay`: 次日首次打开应用时回填前一天的此字段
- `feltPreachy` / `foundHelpful` / `willingToContinue`: 每日结束时弹出简短反馈卡片收集

## 6. 不收集的内容

以下内容**严格禁止收集**，OVERLOAD_RESEARCH_MODE 不得记录、存储或传输:

| 禁止收集项 | 原因 |
|------------|------|
| 真实姓名 | 身份识别信息 |
| 真实签到文本 | 可能包含个人隐私内容 |
| 真实身体状态描述 | 健康敏感信息 |
| 设备序列号 | 硬件标识符 |
| 手机号码 | 个人联系方式 |
| 精确地理位置 | 位置隐私 |
| 通知原始正文 | 第三方应用内容 |
| 照片或截图 | 可能包含隐私信息 |
| 通讯录或社交关系 | 人际关系隐私 |
| 生物特征数据 | 生物识别敏感信息 |

## 7. 知情同意

### 7.1 同意流程
1. 研究者在招募阶段向参与者发放知情同意书（见 `pilot-consent-template.md`）
2. 参与者有至少 48 小时阅读和考虑
3. 参与者签署书面知情同意书后，方可安装试用版本应用
4. 应用首次启动时再次展示数字版知情同意摘要，参与者需点击确认

### 7.2 同意要点
- 数据收集范围和目的已明确告知
- 不收集的内容已明确列出
- 自愿参与原则已说明
- 随时退出权利已告知
- 数据删除权利已告知

---

## 8. 退出机制

### 8.1 参与者主动退出
- 参与者可在试用期间任何时间退出，无需说明理由
- 退出方式: 在应用"隐私"页面点击"清除全部数据"，或直接联系研究者
- 退出后: 设备上所有试用数据将被删除，研究者收到通知后也会删除已导出的该参与者数据
- 退出不影响参与者已获得的补偿（如有）

### 8.2 研究者终止
- 如参与者连续 3 天未使用应用，研究者可发起联系确认
- 如参与者明确表示不愿继续，研究者将终止其参与并删除数据
- 如发现参与者在试用期间出现心理不适，研究者应主动建议退出

### 8.3 退出后数据处理
- 参与者退出后，其已产生的数据应在 7 个工作日内从研究者设备删除
- 删除操作需记录删除日志（仅记录 participantCode 和删除时间，不保留数据本身）
- 参与者可要求获得删除确认

---

## 9. 数据导出和删除方式

### 9.1 数据导出
1. 试用结束后，参与者通过应用内"隐私 > 导出遮挡后的本地数据"功能导出
2. 应用生成符合 `pilot-data-schema.json` 的 JSON 文件
3. 导出文件中取件码、运单号等敏感字段已被掩码处理
4. 参与者通过加密渠道（如加密邮件、面对面传输）将 JSON 文件发送给研究者
5. 研究者收到后确认数据完整性

### 9.2 数据存储
- 研究者收到数据后存储在加密设备上
- 存储期限: 研究报告完成后 90 天内删除
- 最长存储期限: 不超过试用结束后 180 天
- 存储期间不与任何第三方共享

### 9.3 数据删除
- 参与者可随时要求数据删除，无需理由
- 删除请求可通过知情同意书上的联系方式提出
- 研究者在收到请求后 7 个工作日内完成删除
- 删除范围包括: 已导出的 JSON 文件、分析中间产物、备份副本
- 删除后向参与者发送确认通知

### 9.4 数据使用范围
- 仅用于验证 H-001 至 H-010 产品假设
- 研究报告中仅使用聚合数据，不单独呈现任何可识别参与者的数据
- 不用于商业营销、用户画像或其他非研究目的
- 研究报告公开发表时，不包含任何可识别个人的信息
