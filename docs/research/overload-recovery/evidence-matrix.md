## 一、四层证据体系说明

本项目采用四层证据体系，从高到低排列：

| 层级 | 类别 | 代号 | 说明 | 可信度 |
|------|------|------|------|--------|
| 第一层 | 外部学术研究 | A类 | 经过同行评议的学术论文，有明确的研究方法、样本量和统计检验 | 最高 |
| 第二层 | 用户原生声音 | B类 | 来自公开平台的真实用户反馈，包括差评、投诉、社区讨论 | 中高 |
| 第三层 | 竞品分析 | C类 | 对竞争产品的功能、定价、隐私、用户评价的系统分析 | 中 |
| 第四层 | 内部实验数据 | D类 | 产品自身的用户测试、A/B 测试、留存数据（尚未收集） | 待验证 |

**证据使用原则**:
- A类证据用于验证假设的理论基础
- B类证据用于发现问题空间和验证用户痛点
- C类证据用于差异化定位和避免竞品错误
- D类证据用于产品迭代验证（需在 MVP 阶段收集）
- 任何产品决策应至少有 B类以上证据支持

## 三、证据使用规则

### 3.1 引用规范

- 引用学术研究时标注：`(R-XXX, 期刊年份, DOI)`
- 引用用户声音时标注：`(VC-XXX, 来源平台)`
- 引用竞品分析时标注：`(CR-XXX, 产品名称)`
- 多源证据交叉验证时标注：`(R-XXX + VC-XXX + CR-XXX)`

### 3.2 证据权重

| 证据组合 | 权重等级 | 决策建议 |
|----------|----------|----------|
| A + B + C 三类一致 | 极高 | 可直接用于产品决策 |
| A + B 两类一致 | 高 | 可用于产品设计，C类验证后执行 |
| B + C 两类一致 | 中 | 需A类验证或D类实验确认 |
| 仅 A 类 | 中 | 理论支持充分但需用户验证 |
| 仅 B 类 | 低 | 用于假设生成，需进一步验证 |
| 仅 C 类 | 低 | 用于差异化参考，不单独支撑决策 |

### 3.3 更新规则

- A类证据：每季度检索一次新发表的相关论文
- B类证据：每月监控主要平台的用户反馈
- C类证据：每季度更新竞品功能变化
- D类证据：产品上线后持续收集

## 四、关键发现摘要

以下是从收集的证据中提炼的 8 条核心发现：

### 发现 1: 睡前拖延与压力强相关 (r=0.38)，支持"压力驱动"理论

**证据来源**: R-001 (元分析, N=35,097, r=0.38), R-002 (横断面, N=741), VC-004, VC-005, VC-018

**核心结论**: 睡前拖延不是时间管理问题，而是压力驱动的自我调节失败。感知压力与睡前拖延的关联 (r=0.38) 强于抑郁 (r=0.28) 和焦虑 (r=0.30)，说明压力是核心驱动因素。

**产品启示**: 深夜隐藏任务压力（H-002）有理论依据。产品应在夜间降低任务可见性，减少认知负荷。

### 发现 2: 数据输入负担是健康应用弃用的首要原因

**证据来源**: R-007 (全国调查, N=1,604), R-004 (PPI研究, N=118), VC-012, VC-019, VC-021

**核心结论**: 在 Krebs & Duncan 的全国调查中，"高数据输入负担"是用户停止使用健康应用的首要原因。Mair 等人的 PPI 研究也发现"权衡收益与负担"是采用障碍的核心主题。多条用户声音证实"工具本身也成了负担"。

**产品启示**: 十秒签到假设（H-004）有强证据支持。产品必须将每次交互成本降至最低。

### 发现 3: 睡眠追踪评分可能引发焦虑，形成恶性循环

**证据来源**: VC-005, VC-022, R-006 (综述), CR-001, CR-004

**核心结论**: 多条用户声音描述了"看到差评睡眠分数 -> 更加焦虑 -> 更难入睡"的恶性循环。学术综述 (R-006) 也指出睡眠应用可能增加不必要的医疗互动。竞品 Pillow 和 Sleep Cycle 的差评中频繁出现"数据焦虑"。

**产品启示**: 产品不应给出绝对评分（如"睡眠质量 65分"），而应使用非评判性的趋势呈现（如"本周恢复趋势稳定"）。

### 发现 4: 待办列表只增不减导致焦虑-拖延恶性循环

**证据来源**: VC-003, VC-012, VC-019, VC-020, CR-005, CR-008

**核心结论**: 多条用户声音描述了"待办列表越来越长 -> 看到列表就焦虑 -> 焦虑导致拖延 -> 列表更长"的循环。现有任务管理应用（Todoist、Notion）不提供"降载"机制，只允许添加和完成，不支持"隐藏"或"延期"。

**产品启示**: 一键降载假设（H-001）和系统建议延期（H-006）直接针对此问题。产品必须提供"减少可见任务"的机制。

### 发现 5: 心理脱离是恢复的关键机制，未完成任务阻碍脱离

**证据来源**: R-003 (前瞻性纵向, N=337/169, β=.21~.43)

**核心结论**: Blake 等人的前瞻性纵向研究证明，T1 时的心理脱离水平预测 T2 时的心理健康 (β=.21)、焦虑 (β=-.43) 和生活满意度 (β=.32)。未完成的工作承诺会阻碍心理脱离，形成持续负荷。

**产品启示**: Waiting For 形成持续负荷假设（H-003）有间接支持。产品应帮助用户"关闭"工作认知，而非仅管理任务列表。

### 发现 6: 冥想/正念应用坚持率极低，问题在于习惯养成而非内容质量

**证据来源**: R-009 (MARS评估, 23个应用), VC-015, VC-016, VC-017, CR-012, CR-013, CR-014

**核心结论**: 即使是 MARS 评分最高的 Headspace (4.0/5.0)，用户坚持率仍然极低。用户声音显示"身边没有人坚持超过一个月"（VC-017）。问题不在内容质量而在习惯养成机制和与用户当前状态的匹配。

**产品启示**: 冥想应作为恢复手段之一，根据负荷状态智能推荐，而非作为独立产品要求用户"每日冥想"。

### 发现 7: 自动续费和隐性扣费是用户信任的致命杀手

**证据来源**: VC-007, VC-008, VC-009, VC-013, CR-001, CR-002, CR-003, CR-012

**核心结论**: 跨多个产品类别（睡眠、冥想），自动续费和隐性扣费是最集中的用户投诉。蜗牛睡眠和小睡眠因 198 元扣费在黑猫平台被大量投诉。Sleep Cycle 和 Calm 的 App Store 差评中自动续费问题频繁出现。

**产品启示**: 产品必须采用透明的定价模式——免费核心功能 + 一次性付费高级功能，杜绝自动续费陷阱。

### 发现 8: 心理健康应用存在安全治理缺失，需明确非临床边界

**证据来源**: R-010 (系统评估, 69个应用), R-006 (综述)

**核心结论**: Martinengo 等人评估了 69 个抑郁/自杀预防应用，仅 7% 纳入了全部 6 项循证自杀预防策略，9% 提供了错误的危机热线号码。应用商店的自我治理存在系统性失败。

**产品启示**: 产品必须明确非临床定位，不提供自杀风险评估或精神疾病诊断。需要建立安全边界和风险信号引导机制。

## 五、仍需填补的缺口

详见 [evidence-gaps.md](./evidence-gaps.md)

主要缺口包括：
1. 缺乏"一键降载"功能的直接用户验证（H-001, H-006 需真人测试）
2. 缺乏中国用户对健康应用使用习惯的定量研究（现有研究以美国和欧洲为主）
3. 缺乏"负荷-恢复"双轴概念的用户理解度测试
4. 缺乏产品自身的留存和弃用数据（D类证据未开始收集）
5. 缺乏 Waiting For 机制的直接效果验证
