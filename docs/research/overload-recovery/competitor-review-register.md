## 类别一: 睡眠追踪应用

### CR-001: Sleep Cycle

| 字段 | 内容 |
|------|------|
| **名称** | Sleep Cycle |
| **平台** | iOS、Android |
| **核心功能** | 麦克风/加速度计睡眠监测、智能闹钟、睡眠分析报告、打鼾检测 |
| **用户批评** | 监测不准（VC-006）；频繁卡死导致闹钟失效；自动续费取消困难（VC-007） |
| **批评来源URL** | https://apps.apple.com/app/sleep-cycle-alarm-clock/id320606570 |
| **隐私争议** | 需要麦克风权限全程录音；数据上传至云端；隐私政策允许与第三方共享匿名数据 |
| **临床/心理健康声明** | 声称"科学验证的睡眠分析"，但未提供临床验证数据；不声称诊断功能 |
| **与"负荷与恢复助手"的差异化** | Sleep Cycle 专注睡眠数据采集，不关注日间负荷；依赖传感器自动采集而非用户主观感受；睡眠评分可能引发焦虑（VC-005, VC-022）；需要云端上传，不符合本地优先原则 |

## 使用规则

1. 竞品审查基于公开信息（App Store 评论、投诉平台、社区讨论）和学术研究（R-005, R-006, R-009, R-010）。
2. "用户批评"字段引用了用户声音登记表 (VC) 中的具体记录。
3. "与负荷与恢复助手的差异化"字段应直接用于产品设计决策。
4. 竞品分析应定期更新（建议每季度），跟踪竞品功能变化和用户反馈趋势。
5. 隐私争议的评估基于公开隐私政策和技术架构分析，非法律审计。
