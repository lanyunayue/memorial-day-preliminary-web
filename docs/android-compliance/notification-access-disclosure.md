# 通知访问披露

时刻仅在用户进入 Android 系统设置并明确开启通知访问后，才可能收到通知回调。授权本身不等于允许所有来源：来源白名单默认为空，用户还必须逐项开启允许的应用。

白名单来源到达通知时，应用在内存中读取标题和正文以判断是否为快递事件，提取承运商、状态、运单标识、预计时间、取件地点和取件码。完成结构化后立即丢弃原始标题与正文，不写数据库、日志、崩溃报告或网络。

未在白名单的来源在读取 `Notification.extras` 前即被拒绝，不解析、不保存、不记录标题或正文。手动模式完全不读取通知。

应用不申请 `INTERNET`，不上传通知，不控制第三方应用，也不使用 AccessibilityService。
