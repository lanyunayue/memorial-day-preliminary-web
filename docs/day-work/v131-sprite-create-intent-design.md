# v1.3.1 精灵创建意图归一化 - 设计文档

## 模块位置
src/assistant/sprite-create-intent.js

## 导出函数

### normalizeSpriteCreateRequest(text)
主归一化函数，识别创建请求并返回：
- isCreate: boolean - 是否为创建记录意图
- cleaned: string - 剥离命令外壳后的干净文本（保留日期时间）
- sourceText: string - 原始输入完整保留
- hasTime: boolean - 是否包含日期/时间关键词
- verbFound: string|null - 匹配到的创建动词

### extractSpriteCreateIntent(text)
提取创建意图并调用 parser-adapter 生成预览对象：
- title: 事项标题（清理后）
- recordKind: 类型（reminder/note）
- dateKey/timeText/dateText: 时间信息
- isAllDay: 是否全天
- sourceText: 原始输入
- cleanedInput: 归一化输入

### cleanSpriteCreateTitle(text)
从文本中去除日期时间信息，保留事项标题。

## 意图匹配优先级
1. 现有快捷指令（intent-router 中已有的打开日历、查看今天等）
2. 明确创建记录意图（匹配创建动词）
3. 包含日期/时间的事项 parser fallback（如"今天作业还没做"）
4. 无日期备忘 fallback（如"买牛奶"）
5. 真正无法理解才返回 unknown

## 边界条件
- 问句（含？、吗、呢、什么、怎么、为什么等）不作为创建意图
- 闲聊/打招呼（你好、谢谢、测试、随便说点什么）不作为创建意图
- 快捷指令（打开、查看、切换、导出、备份等）优先匹配
- 只有动词没有内容（如"帮我登记"）不视为有效创建
- HTML/script 标签作为文本处理不执行
- 过长输入自动截断
