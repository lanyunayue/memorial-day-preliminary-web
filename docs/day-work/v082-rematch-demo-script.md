# 时刻 v0.8.2 复赛演示脚本

## 一句话定位

时刻不是替代 Days Matter，也不是替代 Google Calendar。它补的是更早一步：把聊天、通知或脑子里的一句话，先用低成本整理成有时间感的记录，再导出到专业日历或做本地备份保护。

## 30 秒演示

1. 打开首页，输入“明天下午三点开会”。
2. 保存后展示记录已经带日期和时间。
3. 进入日历页，说明记录会形成 dot。
4. 进入我的页，展示“导出全部日历文件”和“数据安全”。
5. 结尾说明：重要日程可导出 `.ics` 到系统日历，时刻数据可导出 JSON 备份。

## 1 分钟演示

1. 痛点：很多安排最初不是正式日历事件，而是一句话。
2. 首页输入“明天下午三点开会”，生成提醒记录。
3. 点击“体验示例”，快速生成纪念、提醒、习惯、备忘。
4. 全部页搜索“妈妈”，展示搜索和类型筛选。
5. 日历页展示 dot 和日期记录。
6. 我的页展示：
   - `.ics` 日历导出。
   - JSON 备份。
   - 导入预览。
   - 网页提醒限制说明。

## 3 分钟演示

1. 先讲定位：时刻负责把“一句话”变成“可管理的时间记录”。
2. 首页输入“明天下午三点开会”，展示 NLP 识别日期和时间。
3. 点击“体验示例”，展示 5 条示例记录。
4. 全部页搜索“妈妈”，再筛选纪念、提醒、习惯。
5. 日历页展示 dot，并用快速添加说明它不是静态 demo。
6. 详情页展示单条“导出日历”。
7. 我的页展示批量 `.ics` 导出，并说明这是标准文件手动导入，不是账号同步。
8. 我的页展示“数据安全”：当前记录数、可导出记录数、无日期记录数、上次备份。
9. 整理页导入 JSON，展示导入预览：有效记录、无效记录、缺失 id、重复 id、冲突 id。
10. 说明提醒边界：网页打开时可检查提醒，关闭网页后长期提醒不保证，重要日程建议导出到系统日历。

## 用户旅程

一句话 → 记录 → 日历 dot → `.ics` → JSON 备份。

## 对比 Days Matter

Days Matter 更偏纪念日和倒计时。时刻覆盖提醒、纪念、习惯、备忘，并且入口是一句话整理；它不是只记录一个纪念日，而是把零散安排先变成结构化记录。

## 对比 Google Calendar

Google Calendar 适合正式日程管理。时刻处理的是进入正式日历之前的一步：用户先随手写一句话，确认有价值后再导出 `.ics` 到系统日历。

## 回应 localStorage 不可靠

这是 Web 版的现实边界。v0.8.2 增加了 JSON 备份、导入预览、id 冲突修复、上次备份时间和数据安全提示，降低本地数据丢失风险，但不声称等同云同步。

## 回应网页提醒不可靠

时刻明确说明：网页提醒依赖浏览器环境，页面关闭后长期提醒不一定可靠。重要日程建议导出 `.ics` 到系统日历。

## 时刻精灵

时刻精灵不是宠物系统，也不是游戏化主角。它只是一个轻量入口，帮助首次用户理解“写一句”和“体验示例”。

## 边界说明

当前不支持云同步、登录、数据库、Google Calendar API、后台常驻提醒和跨设备自动同步。这些不是遗漏，而是为了让初版保持可验证、可回滚、低风险。
