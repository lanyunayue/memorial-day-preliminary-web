# v1.3.2 运行态验证报告

**版本**: v1.3.2
**验证日期**: 2026-07-10
**环境**: localhost:8767 (http-server)

## 卡片操作运行态

### 桌面端 (1280×720+，fine pointer)
- 卡片右侧显示 rc-more-btn、rc-edit-btn 等快捷按钮
- 点击 rc-more-btn 弹出绝对定位菜单
- 菜单项：编辑、置顶/取消置顶、复制文本、导出.ics、生成纪念卡、删除
- 点击外部/Esc/切换页面/打开另一张卡时关闭菜单
- 菜单 z-index 高于相邻卡片，不被 overflow 裁剪
- 无 hover 自动展开，布局稳定

### 移动端 (390×844 模拟，coarse pointer)
- swipe-actions 位于卡片背后，flex nowrap
- 横向移动 >16px 且横向 > 纵向时进入 swipe
- 最大 translateX 距离 = action rail 宽度 (240px)
- 松手超过 35% 展开，否则回弹
- pointerdown/pointermove/pointerup 统一处理
- 一次只展开一张卡

### 平板 (768×1024)
- coarse pointer 模式：使用 swipe
- fine pointer 模式：使用 more menu

### 所有视口检查
- 360×640、375×667、390×844、414×896：swipe 正常，按钮不换行
- 768×1024、820×1180：根据 pointer 类型选择模式
- 1024×768、1280×720、1366×768、1440×900、1920×1080：more menu 正常
- 无页面横向 overflow
- 深色/浅色主题样式正确

## Agent 上下文运行态

### 场景验证
1. "我待会还有作业要写" → 生成预览（事项：写作业，日期：今天，时间：未指定（待会）），不写入
2. "晚上八点吧" → 更新同一草稿时间为 20:00
3. "确认" → 写入一条记录，不重复
4. "作业已经写完了" → 不创建新待办
5. "今天有什么" → 调用今日概览，不误判为创建
6. "我今晚还要复习英语" → 生成第二个预览
7. "算了" → 取消草稿，不写入

### 静态测试覆盖
所有 65 项静态测试通过，包括：
- 中文数字时间解析（晚上八点 → 20:00）
- 否定/疑问/已完成/闲聊过滤
- 多轮修改不创建第二条草稿
- "它"引用最近记录
- pending draft 过期机制
- 不静默保存

部署前验证：无应用级 JS error。
